# Architecture

This project keeps the production path small and explicit:

```text
app.py
  -> src.ui.streamlit
  -> src.services.AnswerService
  -> src.generation.AnswerPipeline
  -> optional src.generation.QueryRewriter
  -> src.retrieval.core
  -> src.retrieval.vectorstore / ChromaDB
```

The FastAPI entrypoint uses the same service contract:

```text
src.api.main
  -> src.api.routes.chat
  -> src.services.AnswerService
```

## Module Boundaries

- `src/api`: FastAPI application, routes, schemas, and dependency wiring.
- `src/services`: Shared application service contracts used by UI and API.
- `src/ui/streamlit`: Streamlit UI code.
- `src/generation`: Answer-generation pipeline, optional query rewriting, and guardrails.
- `src/retrieval/core`: Retrieval orchestration, reranking, and structured lookup.
- `src/retrieval/vectorstore`: Embedding and ChromaDB vectorstore access.
- `src/extraction`: Structured extraction from parsed handbook data.
- `src/chunking`: Chunk-building and index manifest generation.
- `src/preprocessing`: Shared preprocessing helpers outside the production runtime path.
- `src/ingestion`: Raw PDF loading.
- `src/common`: Shared cross-cutting utilities such as environment loading.
- `configs`: YAML configuration for pipeline stages.
- `scripts`: Thin local entrypoints for pipeline runners and evaluation scripts.
- `tests`: Offline unit/API tests that should not call Gemini.

## Entity Linking Workflow

Faculty and office names are normalized into `data/processed/entities/entity_registry.json`.
The registry is generated by `src/retrieval/core/build_entity_registry.py` instead
of being edited by hand.

For faculty records, the builder:

- reads `data/processed/directories/faculty_program_directory.json`;
- extracts program headings such as `NGÀNH ...`;
- adds program-name aliases to the parent faculty entity;
- adds accent-folded aliases so accentless Vietnamese can still match;
- keeps manual high-value aliases such as `CNTT`, `KTX`, `PDT`, and `CTSV`.

At runtime, `src/retrieval/core/entity_linker.py` first performs exact normalized
matching, then a conservative fuzzy alias match for light typos. The fuzzy layer
only considers aliases with at least two tokens and uses a high similarity
threshold, so broad one-word aliases do not pull unrelated faculties into the
retrieval set.

When faculty extraction rules change, regenerate the registry and run the entity
tests before rebuilding or deploying:

```bash
python src/retrieval/core/build_entity_registry.py
python -m unittest tests.test_entity_registry_builder tests.test_entity_linker
```
