# API Contract

Base application:

```bash
python -m uvicorn src.api.main:app --reload
```

## Health

```http
GET /health
```

Response:

```json
{
  "status": "ok",
  "service": "student_handbook_rag",
  "version": "0.1.0"
}
```

This endpoint must stay lightweight. It should not load the retrieval pipeline
or call Gemini.

## Artifact Health

```http
GET /health/artifacts
```

Response:

```json
{
  "status": "ok",
  "required_artifacts": [
    {
      "path": "data/vectorstore/chroma",
      "exists": true,
      "kind": "vectorstore"
    }
  ]
}
```

This endpoint checks required config, processed JSON, and ChromaDB vectorstore
paths without loading the embedding model. It is useful after backend deployment
to confirm that mounted artifacts are visible to the API service.

## Chat

```http
POST /chat
```

Request:

```json
{
  "query": "Email Phong Dao tao la gi?",
  "include_debug": false
}
```

Response fields mirror the `AnswerService.answer()` result and keep debug
payloads limited when requested. The API must not duplicate retrieval,
guardrail, citation, cache, or answer-generation logic.

Operational fields:

- `request_id`: per-request identifier generated by the API route.
- `latency_ms`: wall-clock route latency in milliseconds.
- Requests with an empty query return `400`.
- Requests longer than `STUDENT_RAG_MAX_QUERY_CHARS` return `400`. The default
  limit is 500 characters.
- If `STUDENT_RAG_RATE_LIMIT_PER_MINUTE` is greater than 0, the API applies a
  small in-memory per-client rate limit and returns `429` when exceeded.

When `include_debug` is true, the debug payload includes compact routing,
retrieval, citation-count, request ID, and latency fields. It intentionally does
not include the full retrieved context.
