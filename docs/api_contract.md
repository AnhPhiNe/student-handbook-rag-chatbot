# API Contract

Base application:

```bash
python -m uvicorn src.api.main:app --reload
```

## Health

```http
GET /health
```

Response:

```json
{
  "status": "ok",
  "service": "student_handbook_rag",
  "version": "0.1.0"
}
```

This endpoint must stay lightweight. It should not load the retrieval pipeline,
embedding model, vector database, or Gemini.

## Artifact Health

```http
GET /health/artifacts
X-Admin-API-Key: <STUDENT_RAG_ADMIN_API_KEY>
```

This admin-only endpoint checks required config and processed JSON without
loading the embedding model. For local ChromaDB it checks
`data/vectorstore/chroma`; for Qdrant Cloud it checks `QDRANT_URL` and
`QDRANT_API_KEY`. It returns `403` unless `X-Admin-API-Key` matches
`STUDENT_RAG_ADMIN_API_KEY`.

Example Qdrant Cloud response:

```json
{
  "status": "ok",
  "required_artifacts": [
    {"path": "configs/answer_generation.yaml", "exists": true, "kind": "config"},
    {"path": "data/processed/tables/scoring_tables.json", "exists": true, "kind": "processed_json"},
    {"path": "data/processed/tables/formula_rules.json", "exists": true, "kind": "processed_json"},
    {"path": "data/processed/entities/entity_registry.json", "exists": true, "kind": "processed_json"},
    {"path": "data/processed/entities/query_expansion_rules.json", "exists": true, "kind": "processed_json"},
    {"path": "QDRANT_URL", "exists": true, "kind": "env"},
    {"path": "QDRANT_API_KEY", "exists": true, "kind": "env"}
  ]
}
```

## Chat

```http
POST /chat
```

Request:

```json
{
  "query": "Email Phong Dao tao la gi?",
  "include_debug": false,
  "chat_history": []
}
```

`chat_history` is forwarded to `AnswerPipeline`. The API route does not decide
whether history is relevant; the LLM ContextResolver inside the generation layer
does that before QueryRewriter runs.

Response fields mirror `AnswerService.answer()` and keep debug payloads compact
when requested. The API must not duplicate retrieval, guardrail, citation,
cache, or answer-generation logic.

Important response fields:

- `answer`: final answer text, clarification question, or fallback message.
- `status`: `answered`, `needs_clarification`, `low_confidence`,
  `out_of_domain`, `retrieval_error`, or `api_error`.
- `effective_query`: the final query used by the answering pipeline after
  context resolution and safe rewriting.
- `query_rewrite`: compact rewrite metadata. When enabled, it can include
  `context_resolution`, which records whether the current query was treated as
  `standalone_new_topic`, `follow_up`, or `ambiguous`.
- `clarification_needed`: true when `AnswerPipeline` returns a clarification
  question instead of running an unsafe retrieval or answer-generation path.
- `citations_used`: the source citations selected for the answer.

Operational fields:

- `request_id`: per-request identifier generated by the API route.
- `latency_ms`: wall-clock route latency in milliseconds.
- Requests with an empty query return `400`.
- Requests longer than `STUDENT_RAG_MAX_QUERY_CHARS` return `400`; the default
  limit is 500 characters.
- If `STUDENT_RAG_RATE_LIMIT_PER_MINUTE` is greater than 0, the API applies a
  small in-memory per-client rate limit and returns `429` when exceeded. The
  default is 20 requests per minute; set it to `0` only to disable the limit in
  local/dev environments.

When `include_debug` is true, the debug payload includes compact routing,
retrieval, rewrite, citation-count, request ID, and latency fields. It
intentionally does not include the full retrieved context.

## Chat Stream

```http
POST /chat/stream
```

The request body uses the same `ChatRequest` shape as `/chat`. The endpoint
returns Server-Sent Events:

- `metadata`: status, intent, strategy, citations, and request ID.
- `token`: answer text chunks or a single clarification/fallback message.
- `done`: request ID and latency.
- `error`: internal streaming error.

The stream path uses the same empty-query, max-length, and rate-limit controls
as `/chat`.

The stream path uses the same `AnswerPipeline` logic as `/chat`: context
resolution runs before query rewriting, ambiguous context can return a
clarification response, and history-based rewrites are verified before the
answer generator receives retrieved context.
